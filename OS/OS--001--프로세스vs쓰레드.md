### 프로세스와 쓰레드
- 프로세스(Process)란 메모리에 적재되고 CPU의 자원을 할당 받아 프로그램이 실행되고 있는 상태이다.
- 쓰레드(Thread)란 한 프로세스 내에서 동작되는 여러 실행의 흐름이다.
- 쓰레드는 같은 프로세스 안에 있는 여러 쓰레드들은 같은 힙 공간을 공유한다.
***
### 프로세스 vs 쓰레드
- 일반적으로 애플리케이션은 하나 이상의 프로세스를 가지고 있고, 하나의 프로세스는 반드시 하나 이상의 쓰레드를 갖는다.
- 프로세스는 다른 프로세스의 메모리에 직접 접근할 수 없다.
- 쓰레드는 프로세스 내에서 Stack만 따로 할당받고, Code, Data, Heap 영역을 서로 공유한다.
- 프로세스는 자원을 공유하지 않지만, 쓰레드는 자원을 공유한다.
***
### 멀티 프로세스
- 멀티 프로세스란 하나의 애플리케이션을 여러 개의 프로세스로 구성하여 각 프로세스가 하나의 작업을 처리하도록 하는 것이다.
- 여러개의 자식 프로세스 중 하나에 문제가 발생해도, 다른 자식 프로세스에는 영향을 끼치지 않는다.
- 비교적 구현이 간단하고, 각 프로세스들이 독립적으로 동작하여 자원이 서로 다르게 할당된다.
- 메모리 사용량이 많다.
***
### 멀티 쓰레드
- 멀티 쓰레드란 하나의 애플리케이션을 여러 개의 쓰레드로 구성하여 하나의 쓰레드가 하나의 작업을 처리하도록 하는 것이다.
- 프로그램의 일부분이 오류 또는 긴 작업으로 중단되어도 프로그램이 계속 수행된다.
- 자원 공유가 쉬워 쓰레드들은 부모 프로세스의 자원과 메모리를 공유할 수 있다.
- 자식 쓰레드 중 하나에 문제가 생긴 경우, 전체 프로세스에 영향을 끼칠 수 있다.
